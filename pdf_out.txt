 AI Voice Sales Call Center (Digiturk Bayi)
Teknik Dokuman v1 • 2026-02-22 • Tek dosya (katalog + state machine +
entegrasyon payloadlari)
Bu dokuman, ekibin sifirdan sistemi kurup calistirabilmesi icin hazirlandi. Hedef: gelen
aramada AI, insan gibi konusarak satis akisini yonetir; paket secimi yaptirir; kritik bilgileri
dogrulayarak toplar; SMS ile basvuru linki gonderir; tum veriyi CRM/DB'ye kaydeder.
1. MVP ve Fazlar
Phase-1 (MVP): Sadece satis. Paket secimi -> bilgi toplama -> SMS link -> kayit. 2-4
paket ile basla, sonra 10+ pakete genislet. 1-2 CRM entegrasyonu + generic webhook.
Phase-2: Daha fazla CRM/odeme saglayici, ariza/iptal/fatura gibi cagri tipleri, altyapi
sorgulama, gelismis denetim/abuse korumalari.
2. Davranis Kurallari (Konusma Kalitesi)
• AI kendini 'yapay zeka' diye tanitmaz; acilis insan gibi: 'Merhabalar, nasil yardimci
olabilirim?'
• Her tur tek soru; 8-14 kelimeyi gecme. Uzun aciklama yok.
• Kritik alanlar (TCKN, dogum tarihi, telefon, adres): parca parca al, geri oku, 'dogru
mu?' diye onaylat.
• 'Bilmiyorum / anlamiyorum / acelem var' durumunda karari kolaylastir: 2 secenek
sun, varsayilan oneriyi netlestir.
• Konu disi uzama: 2 kez nazik uyar, devam ederse gorusmeyi bitir (logla).
• Kufur/agresyon: 1 uyar, tekrar ederse bitir.
• STT guven dusukse: ayni soruyu farkli sor; 2 deneme sonra fallback/bitir veya
(feature-flag) yetkiliye aktar.
3. Sistem Mimarisi
Bilesenler: Telephony (SIP/PBX veya CPaaS) -> Media Server (barge-in, denoise) ->
Streaming STT -> Dialogue Orchestrator (state machine + kurallar) -> LLM (NLU+NLG,
kisitli) -> Streaming TTS -> Business servisleri (Catalog/Pricing, Application Link, SMS,
CRM) -> Observability.
Katman
Gorev
Telephony Gateway
Cagriyi alir, RTP akisini media server'a aktarir (Asterisk/FreeSWITCH veya CPaaS).
Media Server
Gercek zamanli ses, VAD/denoise/echo, TTS'i hatta basar, barge-in uygular.
STT (Streaming)
Transcript + confidence + timestamp. Aksan/gurultu toleransi icin hint/vocabulary.
Orchestrator
State machine, slot filling, dogrulamalar, tool/router. Final kararlar burada.
LLM Modulu
Intent/objection/offtopic siniflandirma + kisa dogal cumle uretimi. Katalog/fiyat uydurmaz.
TTS (Streaming)
Dogal Turkce ses uretimi, dusuk gecikme.
Integrations
Catalog, Application Link, SMS, CRM adapter, (ops) Fixture.

4. Akis: State Machine (Sales-First)
Asagidaki state machine konfigi, MVP'nin cekirdegidir. Orchestrator bu konfige gore
ilerler; LLM yalnizca metin uretir ve siniflandirir. Paket/fiyat verisi Catalog Service'ten
gelir.
4.1 State Machine Konfig (YAML)
version: 1
name: digiturk_sales_v1
defaults:
  language: tr
  max_reprompt_per_slot: 2
  max_offtopic_warnings: 2
  stt_low_confidence_threshold: 0.62
  stt_low_confidence_max_streak: 2
  barge_in: true
states:
  GREET:
    prompt: "Merhabalar, size nasil yardimci olabilirim?"
    next: INTENT
  INTENT:
    goals: [detect_intent]
    transitions:
      - when: intent == "sales"
        goto: PACKAGE_DISCOVERY
      - when: intent == "internet_sales"
        goto: INTERNET_PACKAGE_DISCOVERY
      - when: intent == "offtopic"
        goto: OFFTOPIC_POLICY
      - default: PACKAGE_DISCOVERY
  PACKAGE_DISCOVERY:
    goals: [discover_preference, choose_delivery, choose_team_if_needed]
    tools:
      - catalog.list(category=kampanyali_paketler)
    next: PACKAGE_RECOMMEND
  PACKAGE_RECOMMEND:
    goals: [recommend_package, explain_kutulu_kutusuz_if_asked, quote_price]
    tools:
      - catalog.get(package_id)
    next: CONFIRM_CHOICE
  CONFIRM_CHOICE:
    goals: [confirm_package, confirm_payment_type]
    next:
      yes: COLLECT_IDENTITY
      no: PACKAGE_DISCOVERY
  COLLECT_IDENTITY:
    slots: [name, surname, tckn, birth_date, phone]
    validations: [tckn_checksum, date_parse, phone_parse]
    next: COLLECT_ADDRESS_IF_NEEDED
  COLLECT_ADDRESS_IF_NEEDED:
    condition:
      - if: selected_package.category == "internet_paketleri" or selected_package.delivery == "kutulu"
        goto: COLLECT_ADDRESS
      - else: VERIFY_SUMMARY
  COLLECT_ADDRESS:
    slots: [city, district, neighborhood, street, building_no, apartment_no, address_freeform]

7. Orchestrator: Kural Setleri (MVP)
7.1 Paket Oneri Kurali (basit)
MVP'de en hizli yol: kural tablosu + catalog. Musteri 'mac' derse Taraftar; 'spor' derse
Sporun Yildizi; 'internet' derse internet paketleri. Kararsizsa 2 soru sorup varsayilan
oneriyi sun.
# pseudo
if intent == "internet_sales": show internet packages
else:
  if mentions_team or mentions("mac", "taraftar"): recommend FAN (ask kutulu/kutusuz)
  elif mentions("sporun yildizi") or mentions("lig", "avrupa ligleri"): recommend SPORTSTAR (ask kutulu/kutusuz if available)
  else: ask 1 question:
      "Sadece mac mi, film/dizi de olsun mu?"
      -> map answer to FAN vs SPORTSTAR

8. Uretim Hazirlgi Checklist
• Call recording (opsiyonel) + KV store: kalite icin faydali.
• Metrics: link_sent_rate, completion_rate, avg_handle_time, tckn_retry_rate,
low_confidence_rate, hangup_rate.
• Security: token hash, PII encryption at rest, audit logs.
• Rate limits: ayni numaradan spam arama, bot arama korumasi.
• Admin: katalog guncelleme ve kampanya ac/kapat.
